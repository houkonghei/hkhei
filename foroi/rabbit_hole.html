故事背景

圍繞著山頂有 $n$ 個山洞，一隻狐狸和一隻兔子住在不同的洞裡。

兔子對狐狸說：

「你從第 $n$ 號洞出發，先到第 1 號洞找我；第二次隔 1 個洞（第 3 號洞）找我；第三次隔 2 個洞（第 6 號洞）找我... 依此類推，如果進了 1000 次洞都找不到我，你就要停下來。」

狐狸滿口答應，找了 1000 次也沒找到兔子，累得再也醒不來。請問兔子可以躲在哪些安全山洞中？

數學規律拆解

  • 1

    山洞編號：$1$ 到 $n$

    在程式中為了方便取餘數（Modulo），我們可以使用 $0$ 到 $n-1$ 表示，最後輸出時再加 1。

  • 2

    狐狸每次進洞的規律：

    第 1 次：第 1 號洞（相當於從 0 走 1 步）

    第 2 次：第 3 號洞（相當於在 1 的基礎上再走 2 步）

    第 3 次：第 6 號洞（相當於在 3 的基礎上再走 3 步）

    第 $i$ 次：在上一次的位置加上 $i$ 步，並對 $n$ 取餘數！

  • 3

    公式表達：

    pos = (pos + i) % n

    注意：若餘數為 0，代表第 $n$ 號洞。

💡 觀察與思考

當 $n = 10$ 時，您會發現不管狐狸找多少次，牠永遠只會進到 1, 3, 5, 6, 8, 10 號洞。而 2, 4, 7, 9 號洞是牠絕對進不去的。

這是因為隨著步數增加，取模運算（$\pmod n$）產生的數列進入了某種週期性循環。在 C++ 程式中，我們可以用一個大小為 $n+1$ 的布林值陣列（Boolean Array）或標記陣列，將狐狸去過的洞標為 `true`，最後把剩下為 `false` 的洞號輸出，這就是最經典的「篩選法」思維！

標準 C++ 程式碼實現

程式採用了直觀且高效率的陣列標記法，完美對應模擬邏輯。

標準：C++11 或以上 競賽標準 O(1000) 複雜度
rabbit.cpp 
// 聰明的兔子 - C++ 完整解決方案
#include <iostream>
#include <vector>

using namespace std;

int main() {
    int n;
    // 讀入山洞數量 n
    if (!(cin >> n)) return 0;

    // visited 陣列用來標記洞號是否被狐狸進去過
    // 陣列大小開 n+1，索引剛好對應 1 到 n 號洞
    // 初始化皆為 false (代表沒進去過)
    vector<bool> visited(n + 1, false);

    int current_pos = 0; // 從第 n 號洞出發，在 modulo n 系統中等同於 0 號洞

    // 狐狸總共找 1000 次
    for (int i = 1; i <= 1000; ++i) {
        // 計算這一次狐狸進洞的編號 (跳躍 i 步)
        current_pos = (current_pos + i) % n;

        // 在餘數為 0 的時候，代表第 n 號洞
        int cave_num = (current_pos == 0) ? n : current_pos;

        // 標記此山洞已被狐狸搜查
        visited[cave_num] = true;
    }

    // 尋找哪些山洞是狐狸「從未進入」的
    bool first = true;
    for (int i = 1; i <= n; ++i) {
        if (!visited[i]) {
            if (!first) {
                cout << " ";
            }
            cout << i;
            first = false;
        }
    }
    cout << endl;

    return 0;
}

關鍵程式邏輯解析

1. 布林值標記陣列 (visited)

我們利用一個大小為 `n + 1` 的布林值向量，在 C++ 裡面表示每個山洞的造訪狀態。初始全部設定為 `false`。

2. 巧妙處理「環形」結構

環形山洞在數學上最適合用取模運算 (Modulo) `%`。從 $0$ 到 $n-1$ 跑， 每跑一圈就會回到開頭。最後當餘數是 `0` 時，我們再手動拉回 `n` 即可，避免了複雜的邊界判斷。

3. 單一空格的分隔格式

在程式尾端，為了配合題目 Output 要求「數與數之間用一個空格分隔」，我們利用一個 `first` 布林旗標來控制是否輸出空格，這是在程式檢測（OJ 系統）中避免多餘空格導致 Presentation Error 的標準做法。

課堂引導與提問推薦

教師在教學時可以問學生：「如果狐狸不是找 1000 次，而是無限次找下去，兔子的安全洞會改變嗎？」

提示學生的數學本質： 當 $n$ 為 $2$ 的冪次方（例如 $2, 4, 8, 16$ 等）時，我們會發現有些特殊的現象。深入引導能讓學生體會「數論」在演算法中扮演的奧妙角色！
